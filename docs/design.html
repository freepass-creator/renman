설계서v5 전기능 × 모던

jpkerp6 — 렌터카 통합 ERP 설계서

목표: jpkerp5의 모든 기능을 빠짐없이 담되, 현행 트렌드 UX로. · 독립 데이터모델(v5는 ETL 이관) · 멀티테넌트(운영자/위탁사) · 2026-06

v6는 v5 기능의 복제가 아니라 재구성입니다. v5가 페이지마다 흩어 두었던 계산·상태·디자인을 한 곳(엔진·키트)에 모으고, 사람은 사실만 넣으면 상태·미수·장부가·리스크는 엔진이 파생합니다. 화면은 그 결과를 얇게 보여줄 뿐입니다.

★제품 전략 — 점진적 채택 & 마이그레이션 우선

모든 업체에 풀 ERP를 강요하지 않는다. 베이스 2개(운영현황·리스크)만으로 엑셀을 대체하고, 더 깊게 쓰고 싶은 업체만 심화 도메인으로 내려간다. 진입장벽을 최소화해 “필요에 의해” 자연스럽게 ERP로 들어오게 만든다.

티어구성누가·무엇
베이스
필수·누구나		 운영현황 + 리스크 최소 레코드만으로 “누가 어떤 차를 언제~언제, 얼마에, 몇 세로 쓰는지” 관리 + 안 맞으면 경고. 이 2개만 써도 충분.
심화
필요한 업체만		 자산 · 계약 · 재무 · 일반 자산=차량 생애주기 전부(등록증·스펙·수익·증권·할부·정비/수리/비용) · 계약=계약의 모든것 · 재무=재무의 모든것 · 일반=회사의 모든것. 사실을 더 넣으면 자동 활성.

베이스 최소 레코드 (한 줄 = 한 대여)

  • 차량번호 · 임차인(누가)
  • 계약 시작일 ~ 종료일
  • 월 금액 · 입금액(낸 돈)
  • 운전자 연령 · 보험 허용연령

그 외 모든 필드는 optional. 베이스만 채워도 운영현황·리스크가 완전히 동작해야 한다.

리스크 = 정합성 경고 3종 (베이스의 가치)

  • 미수 — 월금액 × 경과 − 입금 > 0 (돈 내야 하는데 안 냄)
  • 보험 불일치 — 운전자 연령 < 보험 허용연령 (보험 안 맞음)
  • 반납 지남 — 오늘 > 종료일 · 미반납

전부 베이스 레코드에서 파생. 추가 입력 없이 경고가 뜬다.

마이그레이션 우선 — 엑셀 1시트로 즉시 전환. 기존 엑셀로 관리하던 업체가 베이스 컬럼(차량번호·임차인·시작·종료·월금액·입금·운전자연령·보험연령) 1시트를 업로드하면 곧바로 운영현황 + 리스크 경고가 살아난다. 별도 학습·셋업 없이 “기본값만으로” 전환. → 이게 v6의 첫 사용자 경험이자 GTM.

06빌드 순서 — 베이스 → 수직 슬라이스 → 복제

베이스: 최소레코드 + 엑셀 마이그레이션 + 운영현황 + 리스크 경고3종

먼저 베이스(엑셀 1시트 → 운영현황 + 리스크) 를 완성 = 엑셀 쓰던 업체가 바로 전환 가능한 최소 제품. 그 위에 심화 슬라이스를 깊게:

계약→차량→수납→리스크 ⇒한 바퀴 실데이터 데모

한 줄기를 실데이터로 깊게 끝내 패턴(엔진·드로어·키트)을 확정한 뒤, 나머지 도메인은 같은 패턴 복제. 도메인마다 루프: ① 파생 엔진(*-ops.ts) 작성 → 헤드리스 검증 → ② 얇은 목록 + 공통 드로어 연결.

데모 성공 기준: 계약 1건을 인도→운행→은행입금 자동매칭→반납(일할 자동)→미납 시 리스크 등장까지 클릭하면, 같은 사실에서 운영·차량·수납·리스크 4화면이 동시 자동 갱신.

복제 순서(슬라이스 후)

  1. 공통 드로어 전 목록 확산 (운영·자산·손님·보험·과태료)
  2. 수납 매칭 실데이터화 (은행 → 회차 FIFO, 감사)
  3. 리스크 엔진 (aging·시동제어·만기 통합) + 채권/내용증명 워크플로
  4. 자산 서브도메인 8종 (탭/드로어)
  5. 마스터(직원/거래처/코드) · 감사로그 · RBAC · 회계마감
  6. PDF(계약서·내용증명·정산서) · SMS · 첨부 · ⌘K · 반응형 모바일

07크로스커팅 (처음부터 박는 것)

멀티테넌트

운영자=합본, 위탁사=자기회사. 회사 태그 + Firestore Rules 서버 강제. 회사별 코드 scoped.

RBAC

역할: 운영자/위탁사 + 직원(Master/Staff). 라우트·액션·드로어 버튼 게이팅.

감사 · 소프트삭제

create/update/delete/restore 추적(누가·언제·무엇) + 휴지통 + OCR 원본 영구보존.

회계 마감 · 백업

월/분기/연 기간 잠금(마감 후 변경 차단) + Firestore 자동 백업.

08로드맵 & 투자/B2C 트랙

  • Phase 0 — 베이스 + 마이그레이션: 최소레코드 · 엑셀 1시트 업로드 · 운영현황 · 리스크 경고3종. 엑셀 대체 MVP(누구나 즉시 전환)
  • Phase A — 수직 슬라이스: 계약→차량→수납→리스크 실데이터 데모 (투자용 “살아있는 한 바퀴”)
  • Phase B — 전 도메인 복제: 자산 서브·재무 GL·과태료 PDF·마스터·감사·RBAC·마감
  • Phase C — 연동: SMS·PDF·파일첨부·⌘K·반응형 모바일·v5 데이터 ETL 이관
  • Phase D — 플랫폼: GPS/시동제어 실연동 · 렌터카 예약 B2C ERP 확장
투자 내러티브. “데이터 한 곳 → 실시간 파생 → 회사별 격리 → 전부 추적 → 모바일·B2C로 확장.” 슬라이스 한 바퀴가 이 서사를 작동하는 데모로 증명한다.
